## DEC-001: MBTI移出综合评分体系 (2026-06-17)

### 背景

V0.1-V1.1版本中，MBTI被用于计算”外向水平”维度分数，并纳入五维综合竞争力和岗位匹配度计算。

### 问题

1. **科学性不足**：MBTI外向水平评分（E型65-90分，I型35-60分）对内向型求职者存在系统性不利影响，可能导致匹配度偏低
2. **岗位判断偏差**：MBTI不应直接决定用户是否适合某个岗位，这与产品”帮助求职者”的初衷矛盾
3. **用户反馈风险**：用户可能因MBTI评分低而对自己的职业选择产生不必要的怀疑

### 决策

- **移除**：MBTI/外向水平从综合竞争力评分和岗位匹配度中移除
- **保留**：MBTI作为辅助画像信息，在结果页提供人格特征参考
- **新增**：人格特征参考卡片，展示沟通风格、学习偏好和职业发展建议
- **四维评分**：综合竞争力仅由学历背景、项目经验、专业技能、外语水平四个维度计算

### 影响范围

- 岗位模型：移除extroversion维度要求
- 雷达图：五维→四维
- 评分解析弹窗：更新为四维标准
- 匹配度计算：四维算术平均
- 缺口分析/提升建议：移除”外向水平”维度

### 验收标准

- [x]  综合评分不包含MBTI维度
- [x]  匹配度计算不包含外向水平
- [x]  MBTI选择后显示人格特征参考卡片
- [x]  卡片明确标注”不作为岗位匹配评判依据”
- [x]  未选择MBTI时不显示人格特征参考卡片

## DEC-000: 初始技术选型 (2026-06-16)

### 决策

- 纯前端实现（HTML+CSS+JS），无外部依赖
- 评分逻辑使用关键词匹配模拟（Demo阶段）
- 后续迭代保留纯前端版本用于作品集展示，另行开发接入真实AI的版本

### 理由

- 降低部署门槛，浏览器直接打开即可运行
- 作品集展示不依赖后端服务
- Demo阶段重点验证产品逻辑而非技术实现
